Bienvenue dans la documentation de Juju Edition Sphinx¶

Contenu

Introduction¶

Ce tutoriel vous montrera comment commencer avec Juju, en incluant l’installation, la configuration, et le bootstrap d’un nouvel environnement Juju. Avant de commencer vous aurez besoin de :

  • Ubuntu, OSX ou un machine Windows pour y installer le client.
  • Un environnement qui peut fournir un nouveau serveur avec une image du système d’exploitation Ubuntu pour nuage sur demande. Cela comprend les services tels que Amazon EC2, HP Cloud, une installation d’OpenStack, ou votre machine locale
  • Une paire de clés SSH. Si vous n’en n’avez pas déjà mis un en place, vous devez générer une paire de clés appropriées. Sous Linux et Mac OSX: ssh-keygen-t rsa-b 2048 Sous Windows: Voir les instructions Windows pour SSH et PuTTY _

Installation¶

Pour installer Juju, il vous suffit de récupérer le dernier paquet juju-core du PPA:

  • **Ubuntu : **

    sudo add-apt-repository ppa:juju/stable
sudo apt-get update && sudo apt-get install juju-core

  • ** Mac OSX : **

    Juju est dans Homebrew _, pour l’installer faire :

    brew install juju

  • ** De Windows: **

    Téléchargez et exécutez le Juju installateur de fenêtres d’ici. _

Configuration¶

Maintenant, le logiciel Juju est installé, il doit être configuré pour utiliser le fournisseur de nuage particulier. Cela se fait par la production et l’édition d’un fichier, environments.yaml, qui vivra dans votre juju ~ /. Répertoire / sous Linux et OSX, % LOCALAPPDATA% / Juju sur Windows. Vous pouvez générer le fichier environnements manuellement, mais Juju comprend également une option de configuration passe-partout qui étoffer plus du fichier pour vous et de minimiser la quantité de travail (et les erreurs potentielles).

Pour générer un fichier de configuration initial, il vous suffit de lancer:

juju generate-config

Cette commande provoquera un fichier à écrire dans votre répertoire de juju ~ /. S’il n’existe pas déjà un fichier environments.yaml. Il permettra également de créer le répertoire ~. / Juju si cela n’existe pas.

Ce fichier contient les profils échantillons pour différents types de services de cloud computing, mais vous devez modifier les fichiers de fournir des informations spécifiques à votre fournisseur de cloud. Les articles sont créés pour Amazon Les Services (AWS), HPCloud et une instance de OpenStack générique (et pour le fournisseur local si vous travaillez sur Linux). Pour plus de détails sur ce qui doit être changé, voir les sections pertinentes ci-dessous.

Test de votre configuration¶

Une fois que vous avez installé et configuré Juju, il est probablement une bonne idée de prendre un peu d’un essai et vérifier que tout fonctionne comme prévu. Parce que Juju, il est vraiment facile à déployer des services, c’est en fait simple et rapide.

La première chose à faire est de mettre en place un environnement de démarrage. Ceci est un exemple dans le nuage que Juju va utiliser pour déployer et contrôler d’autres services avec. Il sera créé selon la configuration que vous avez fourni, et votre clé SSH sera automatiquement téléchargé afin que Juju peut communiquer en toute sécurité avec l’instance de démarrage.

juju bootstrap

Remarque: Si vous disposez de plusieurs environnements configurés, vous pouvez choisir celui qui pour répondre à une commande particulière en ajoutant le commutateur -e suivi par le nom de l’environnement, par exemple -e hpcloud.

Vous pourriez avoir à attendre quelques instants pour cette commande pour revenir, car il a besoin pour effectuer diverses tâches et contactez votre fournisseur de cloud.

En supposant qu’il renvoie avec succès (voir par ailleurs les messages d’erreur courants et quoi faire à leur sujet), nous pouvons désormais déployer des services et d’explorer les opérations de base de Juju.

Pour commencer, nous allons déployer Wordpress, en exécutant cette commande:

juju deploy wordpress

Juju vont maintenant chercher le charme Wordpress et l’utiliser, par exemple le bootstrap à la demande et de déployer toutes les ressources dont elle a besoin pour mettre en place ce service.

Wordpress a besoin d’une base de données, donc nous allons également déployer un de ceux:

juju deploy mysql

Une fois de plus, juju feront tout ce qui est nécessaire pour déployer ce service pour vous, et il peut prendre un certain temps pour la commande pour revenir.

Remarque: Si vous souhaitez obtenir plus d’informations sur ce qui se passe, ou pour aider à résoudre les problèmes de, vous pouvez ajouter l’option-v à la commande de juju pour obtenir une sortie détaillée.

Bien que nous ayons déployé Wordpress et une base de données MySQL, ils ne sont pas encore reliés entre eux en aucune façon. Pour ce faire, nous devons exécuter:

juju add-relation wordpress mysql

Cette commande utilise les informations fournies par les charmes pertinents à associer ces services à l’autre de quelque manière logique. Il ya beaucoup plus à dire sur la liaison des services ensemble qui est couvert dans la documentation de la commande de juju, mais pour le moment, nous avons juste besoin de savoir ce qu’il lie ces services ensemble.

Afin de rendre notre Wordpress publique, nous devons maintenant exposer ce service:

juju expose wordpress

Ce service va maintenant être configuré pour répondre aux requêtes web, afin que les visiteurs peuvent voir. Mais où est-il exactement? Si nous lançons la commande d’état de juju, nous serons en mesure de voir quels services sont en cours d’exécution, et où ils se trouvent.

juju status

Il est tout à fait beaucoup d’informations ici. la première section, machines intitulé:, détaille toutes les instances qui sont en cours d’exécution. Pour chaque, vous verrez la version de Juju ils sont en cours d’exécution, leur hôte, ID instance et la série ou la version de Ubuntu ils sont en cours d’exécution.

Après cela, les sections répertorient les services qui sont actuellement déployés. Les informations ici diffère légèrement selon le service et comment il est configuré. Il sera toutefois, la liste toujours le charme qui a été utilisé pour déployer le service, s’il est exposé ou non, et quelle que soit son adresse relations existent.

De ce statut lecture, nous pouvons voir que wordpress est exposé et prêt. Si nous copions simplement l’adresse dans un navigateur Web, nous devrions être en mesure de le voir courir

Félicitations, vous venez de déployer un service avec Juju!

Maintenant, vous êtes prêt à déployer n’importe quel service vous voulez vraiment des 100s disponibles à l’Juju Charm magasin. _

Pour supprimer tous les déploiements actuels et dissiper tout sur votre nuage, vous pouvez exécuter la commande:

juju destroy-environment

Cela permettra d’éliminer tout, y compris le nœud de démarrage.

Pour en savoir plus sur les charmes, y compris les options de configuration et la gestion des systèmes en cours d’exécution, vous devez continuer à lire la documentation de charme. _
